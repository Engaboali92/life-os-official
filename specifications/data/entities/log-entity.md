# Log Entity

**Document ID:** LOS-DAT-ENT-005

**Status:** Draft

**Owner:** Data Team

---

# Purpose

The Log entity represents an immutable historical record of actions performed within Life OS.

It serves as the canonical source for activity history, auditing, reporting, and analytics.

---

# Responsibilities

- Store immutable activity records.
- Record execution timestamps.
- Support auditing.
- Support reporting.
- Support historical analytics.

---

# Attributes

| Name | Type | Required |
|------|------|----------|
| id | UUID | Yes |
| userId | UUID | Yes |
| entityType | String | Yes |
| entityId | UUID | Yes |
| action | String | Yes |
| value | JSON | No |
| source | String | Yes |
| createdAt | Timestamp | Yes |

---

# Relationships

- Belongs to User
- References Business Entities
- Supports Reports
- Supports Analytics

---

# Constraints

- Identifier is immutable.
- Log records are append-only.
- Log records cannot be modified.
- Log records cannot be deleted except through archive policy.

---

# Validation Rules

- Required fields cannot be empty.
- Entity type must be supported.
- Action must be supported.
- Timestamp generated by the system.

# Security

- Accessible only by the owning user.
- Audit logging is mandatory.
- Records are immutable.
