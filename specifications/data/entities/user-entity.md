# User Entity

**Document ID:** LOS-DAT-ENT-001

**Status:** Draft

**Owner:** Data Team

---

# Purpose

The User entity represents the owner of all personal information managed by Life OS.

Every business record belongs to exactly one user.

---

# Responsibilities

- Store user identity.
- Own all personal records.
- Provide ownership boundaries.
- Support authorization.
- Support auditing.

---

# Attributes

| Name | Type | Required |
|------|------|----------|
| id | UUID | Yes |
| displayName | String | Yes |
| email | String | Yes |
| timezone | String | Yes |
| locale | String | Yes |
| createdAt | Timestamp | Yes |
| updatedAt | Timestamp | Yes |
| archivedAt | Timestamp | No |

---

# Relationships

- Owns Habits
- Owns Tasks
- Owns Goals
- Owns Worship records
- Owns Health records
- Owns Finance records
- Owns Learning records
- Owns Family records
- Owns Work records

---

# Constraints

- Identifier is immutable.
- Email must be unique.
- Timezone must be valid.
- Locale must be supported.

---

# Validation Rules

- Required fields cannot be empty.
- Email must be valid.
- Timestamps are generated by the system.

---

# Lifecycle

Created

↓

Active

↓

Archived

---

# Security

- Accessible only by the owning user.
- Identifier never changes.
- Audit logging required.
