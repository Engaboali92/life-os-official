# Fitness Entity

**Document ID:** LOS-DAT-ENT-013

**Status:** Draft

**Owner:** Data Team

---

# Purpose

The Fitness Entity represents workouts, exercises, body measurements, cardio sessions, and fitness progress managed by Life OS.

---

# Responsibilities

- Store workout records.
- Track exercises.
- Track cardio sessions.
- Track body measurements.
- Support fitness analytics.

---

# Attributes

| Name | Type | Required |
|------|------|----------|
| id | UUID | Yes |
| userId | UUID | Yes |
| activityType | Enum | Yes |
| title | String | Yes |
| duration | Integer | No |
| calories | Decimal | No |
| status | Enum | Yes |
| notes | String | No |
| occurredAt | Timestamp | Yes |
| createdAt | Timestamp | Yes |
| updatedAt | Timestamp | Yes |
| archivedAt | Timestamp | No |

---

# Relationships

- Belongs to User
- Produces Fitness Reports
- Produces Fitness Statistics

---

# Constraints

- Identifier immutable.
- Activity type required.
- Archived records are read-only.

---

# Validation Rules

- Required fields cannot be empty.
- Duration must be zero or greater.
- Calories must be zero or greater.
- Timestamp generated by the system.
