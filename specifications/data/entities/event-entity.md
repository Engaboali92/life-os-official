# Event Entity

**Document ID:** LOS-DAT-ENT-006

**Status:** Draft

**Owner:** Data Team

---

# Purpose

The Event entity represents a domain event emitted by the business layers of Life OS.

It serves as the canonical source for asynchronous processing, auditing, integrations, and event-driven workflows.

---

# Responsibilities

- Store domain events.
- Preserve event history.
- Support event replay.
- Support automation.
- Support integrations.

---

# Attributes

| Name | Type | Required |
|------|------|----------|
| id | UUID | Yes |
| eventType | String | Yes |
| aggregateType | String | Yes |
| aggregateId | UUID | Yes |
| userId | UUID | Yes |
| payload | JSON | Yes |
| version | Integer | Yes |
| source | String | Yes |
| correlationId | UUID | No |
| causationId | UUID | No |
| createdAt | Timestamp | Yes |

---

# Relationships

- Belongs to User
- References Business Entities
- Triggers Automation
- Supports Integrations
- Supports Audit History

---

# Constraints

- Identifier is immutable.
- Events are append-only.
- Payload cannot be modified.
- Version must be positive.

---

# Validation Rules

- Required fields cannot be empty.
- Event type must be registered.
- Aggregate identifier must exist.
- Payload must be valid JSON.
- Timestamp generated by the system.
